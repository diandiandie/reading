# 小学生读书打卡工具

这是一个面向家长使用的响应式读书打卡工具原型，入口文件是 `reading-checkin.html`。当前版本使用浏览器 `localStorage` 保存数据，适合先验证家庭读书计划、打卡和奖励兑换流程；后续部署给多个家庭长期使用时，建议替换为后端账号、数据库和对象存储。

## 当前功能

- 手机号和密码登录，登录后只读取账号绑定家庭的数据。
- 登录页默认只填写手机号和密码；新用户需切换到注册，填写家庭名称、家长称呼和默认孩子。
- 支持一个家庭一个家长账号，也可以扩展为同一家庭多位家长账号。
- 添加自己家的孩子，并设置一个默认孩子。
- 为某个孩子创建读书计划或任务，可填写书本、目标次数、日期、默认奖励金额和备注。
- 在计划内打卡，记录日期、孩子、阅读时长、奖励金额、照片和备注。
- 奖励汇总按孩子分别统计，自动计算每个孩子的累计奖励、已兑换、可兑换余额。
- 兑换记录，支持日期、孩子、兑换金额和备注。
- 手机端和 PC 端自适应布局。

## 本地运行

```bash
pip install -r requirements.txt
python run.py
```

然后访问：

```text
http://127.0.0.1:8000/reading-checkin.html
```

新用户注册：

1. 点击登录页的“注册”。
2. 填写手机号、密码、家庭名称、家长称呼、默认孩子。
3. 点击“注册并进入”。

## 后续服务端化建议

更完整的部署前资料已整理到：

- [数据库设计](docs/database-design.md)
- [后端接口草案](docs/api-design.md)
- [部署前清单](docs/deployment-checklist.md)

建议后端至少包含这些表或集合：

- `families` 或 `organizations`: 家庭信息、家庭名称、状态。
- `users`: 登录账号、姓名、手机号或邮箱、密码哈希。
- `children`: 孩子资料、默认孩子标记、所属家庭。
- `family_users`: 家庭与家长用户的绑定关系。
- `plans`: 读书计划、书名、目标、奖励规则、所属家庭和孩子。
- `checkins`: 打卡记录、日期、奖励金额、备注、所属家庭、计划和孩子。
- `checkin_photos`: 打卡照片 URL、存储对象 key、所属打卡记录。
- `redemptions`: 奖励兑换流水、日期、金额、备注、所属家庭和孩子。

服务端接口需要始终以当前登录用户所属的 `family_id` 作为查询条件，避免跨家庭读取或修改数据。

## 推荐确认顺序

1. 先用当前页面确认家庭注册、登录、孩子、计划、打卡、奖励兑换流程。
2. 确认字段是否够用，例如孩子生日、头像、计划结束条件、奖励是否允许透支。
3. 按数据库设计建表。
4. 按接口草案实现后端。
5. 把当前前端从 `localStorage` 改为请求后端接口。
6. 做跨家庭数据隔离和照片上传测试后再部署正式环境。
